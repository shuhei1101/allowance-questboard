# デモフォルダ整理案 - ハイブリッド構成

## 現状の課題00
- demoフォルダ内のページが混在している
- srcのfeaturesと構成が異なり一貫性がない
- 開発支援機能とビジネス機能が混在している

## 整理方針
- srcのfeaturesと同じ構成でビジネス機能を配置
- 開発支援機能は別階層（development/）に配置
- 機能ごとの関心の分離を明確にする

1. **一貫性**: features配下はsrcと同じ構成

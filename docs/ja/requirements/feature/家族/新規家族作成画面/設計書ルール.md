# 設計書について

## 命名規則
- 設計書ではクラス名やプロパティ名は論理名(日本語)で書くこと
  - 例: `family_members`テーブルの`family_member_id`は、論理名では「家族メンバーID」とする
  - 実装時に英語に変換すること

- 名詞+動詞系の英語は動詞を先頭にすること
  - 家族選択ダイアログ→ `SelectFamilyDialog`。❌️: `FamilySelectionDialog`

### 値オブジェクトについて
- ドメイン固有の値は値オブジェクトを参照もしくは作成してください。
  - 配置場所: [配置場所](../shared/共有_値オブジェクト.md)

- 設計書では、プリミティブの型は日本語で書くこと
  - 例: `string`は「文字列」、`number`は「数値」とする
  - 型が値オブジェクトなど、関連するドキュメントが存在する場合は、リンクを貼ること
    - 例:
    ```markdown
    - ユーザID: [ユーザID](../shared/共有_値オブジェクト.md#ユーザID)
    ```

- 命名はバックエンドのエンティティを参考にすること
  - packages/backend/src/features/{関心事名}/entity/{エンティティ名}.ts
